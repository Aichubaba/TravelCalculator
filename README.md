## Общие сведения

TravelCalculator — одностраничное веб-приложение (SPA), предназначенное для планирования автомобильных путешествий. Программа позволяет:

- строить маршруты по заданным точкам на интерактивной карте;
- рассчитывать полную стоимость поездки с детализацией по категориям (проживание, питание, топливо, дополнительные расходы);
- автоматически конвертировать валюты с использованием актуальных курсов;
- определять время в пути с учетом остановок и ночевок;
- сохранять все данные в локальном хранилище браузера для продолжения работы после перезагрузки.

## Функциональные возможности

- **Интерактивная карта** — визуализация маршрута и точек на основе Leaflet.
- **Построение маршрута** — определение оптимального пути через все активные точки с помощью сервиса OSRM.
- **Калькулятор бюджета** — автоматический пересчет при изменении любых параметров; поддержка ежедневных и фиксированных расходов.
- **Конвертер валют** — работа с RUB, USD, EUR; обновление курсов каждые 6 часов через внешний API.
- **Расчет транспортных затрат** — автоматическое определение стоимости топлива на основе типа автомобиля, его расхода и текущих цен.
- **Расчет времени в пути** — учет кратковременных остановок (каждые 2 часа) и длительных ночевок (после 8 часов движения).
- **Управление точками маршрута** — drag-and-drop для изменения порядка, удаление, переименование, временное исключение.
- **Поиск адресов** — геокодирование через Nominatim (OpenStreetMap) для добавления точек по текстовому запросу.
- **Темная тема** — переключение между светлым и темным оформлением.
- **Локализация** — русский и английский языки интерфейса.
- **Сохранение состояния** — автоматическая синхронизация данных с localStorage.

## Архитектура приложения

Приложение построено по компонентному принципу с четким разделением на слои:

1. **Слой представления (components)** — React-компоненты, отвечающие за отображение данных и обработку пользовательских действий.
2. **Слой бизнес-логики (hooks)** — кастомные хуки, инкапсулирующие вычислительные алгоритмы (расчет бюджета, маршрута, конвертация).
3. **Слой сервисов (services)** — функции для взаимодействия с внешними API (OSRM, Nominatim, Exchange Rate API, цены на топливо).
4. **Слой утилит (utils)** — чистые функции для математических расчетов (гаверсинус, время в пути, бюджет).
5. **Слой данных (data)** — статические данные, например, справочник цен на топливо.

**Поток данных:**
1. Пользователь вводит данные в интерфейсе (добавляет точки, меняет параметры).
2. Компоненты вызывают переданные callback-функции, обновляя состояние в корневом компоненте App.
3. App через хуки передает новые данные в `useRoute` и `useBudgetCalculator`.
4. Хуки при необходимости обращаются к сервисам (OSRM, курсы валют) и утилитам.
5. Результаты вычислений возвращаются в App и передаются вниз для обновления интерфейса.

**Управление состоянием** реализовано через `useState` и кастомный хук `useLocalStorage`, который синхронизирует ключевые объекты (`trip`, `expenses`, `settings`) с локальным хранилищем.

## Установка и запуск

### Требования
- Node.js не ниже версии 16
- npm или yarn

### Инструкция

1. Клонирование репозитория:
   ```bash
   git clone <url-репозитория>
   cd TravelCalculator
   ```

2. Установка зависимостей:
   ```bash
   npm install
   ```

3. Запуск в режиме разработки:
   ```bash
   npm run dev
   ```
   Приложение будет доступно по адресу `http://localhost:5173`.

4. Сборка для production:
   ```bash
   npm run build
   ```

5. Предпросмотр production-сборки:
   ```bash
   npm run preview
   ```

## Взаимодействие с внешними сервисами

**OSRM (Open Source Routing Machine)**
- Назначение: построение автомобильных маршрутов.
- Формат запроса: GET `http://router.project-osrm.org/route/v1/driving/{lon1},{lat1};{lon2},{lat2}?overview=full`
- Ответ: JSON с полями `distance` (метры), `duration` (секунды), `geometry` (закодированная полилиния).

**Nominatim (OpenStreetMap)**
- Назначение: прямое и обратное геокодирование.
- Используется компонентом `AddressSearch` для преобразования текстового адреса в координаты.

**Exchange Rate API**
- URL: `https://api.exchangerate-api.com/v4/latest/RUB`
- Назначение: получение актуальных курсов валют.
- Обновление: каждые 6 часов при запущенном приложении.
- При недоступности сервиса используются последние сохраненные курсы из состояния.

**Цены на топливо**
- Источник: локальный JSON-файл (`src/data/fuelPricesRU.json`).
- Автоматически подставляется при выборе типа топлива в форме `TripForm`.
